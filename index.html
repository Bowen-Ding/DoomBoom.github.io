Hadoop集群搭建

安装一台vmware虚拟机

  • 系统版本: centos7.0

  1. 关闭防火墙和SELINUX

    务必确保防火墙和selinux关闭,否则会导致许多问题

  2. 设置静态ip

    核心是需要保持这几个地方一致:

    1. 宿主机适配器设置中VMnet8属性中IPv4属性设置:
      • 不自动获取IP地址
      • ip地址和网关在同一网段,这里我设置为192.168.13.101
      • 默认网关与首选DNS相同
    2. 虚拟机中网关与宿主机VMnet8默认网关相同,自定义IP处于同一网段,即都为1中示例的192.168.13.X,首选DNS与网关相同

    2.1 具体设置方法

    • 编辑配置ifcfg-ens33,根据网卡名称,文件名会有变化,可通过ip addr查看
      vi /etc/sysconfig/network-scripts/ifcfg-ens33

    • 编辑配置network
      vi /etc/sysconfig/network

  3. 新建用户组及赋予root权限

    3.1 新建用户组

    • 将hadoop用户加入用户组: usermod -G hadoop hadoop
    • 重置用户密码:``passwd hadoop`

    3.2 赋予root权限

    • 编辑文件/etc/sudoers

      在root ALL=(ALL) ALL下添加一行:hadoop ALL=(ALL) ALL

      hadoop用户若想用root权限执行命令,需要在命令前添加’sudo’,例如sudo touch a.txt

  4. 克隆虚拟机并修改初始配置

    a. 通过vmware克隆为3台虚拟机

    b. 开启这三台虚拟机:

    • 修改/etc/sysconfig/network-scripts/ifcfg-ens3文件,同一网段内,更改ip地址,保证ip不冲突,重启网卡;
      service network restart
    • 修改主机名hostnamectl set -hostname XXX

Hexo搭建个人博客及多终端同步使用

  1. 前言

    前期准备:

    a. git账号及git for windows客户端

    b. node.js安装并添加到系统环境变量

  2. 搭建github博客

    2.1 创建仓库

    ​ 新建一个名为 git用户名.github.io 的仓库,这也是博客的访问地址

    2.2 绑定域名

    ​ a. 先到云服务商如阿里云购买一个域名,根据域名带不带www配置方式也会有所不同

    ​ b. 若不带www,需要先ping一下git用户名.github.io 的ip,然后到域名DNS设置页,将A记录指向你ping出来的IP，将CNAME指向git用户名.github.io，这样可以保证无论是否添加www都可以访问.

    ​ c. 然后到github项目根目录新建一个名为CNAME的文件，文件内容填写上域名

    ​ 注意: git用户名.github.io 这个地址仍然可以访问,将会跳转到域名

  3. 配置SSH public key

    a. 打开git客户端的git bash,执行ssh-keygen

    b. cat ~/.ssh/id_rsa.pub 命令查看public key,登录github,依次进入settings -> SSH and GPG keys -> new SSH key,然后将public key粘贴进去
